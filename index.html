Building Blender

  • On Linux

  • On Windows

Overview

This section is only to list the main steps on all platforms, for anyone unfamiliar with the process of compiling C/C++ software from source.

  • Install dependencies(git, subversion, C/C++ compiler)
  • Checkout Blender's source code from version control
  • Use CMake to compile the source code into an executable

Exact steps depend on your OS. Please see links above.

Resolving Build Failures.

Below are some common issues that can result in build errors:

  • Missing dependencies

    On windows and OS X, we provide dependencies.
    Before troubleshooting make sure that your local "lib/" file has been updated.

  • Local changes

    Sometimes, changes made to the code (patches, edits when developing, etc...) can lead to build errors.

    Before spending too much time investigating an error, check that your checkout has no local changes.
    You can stash your local changes, and restore them later using the git stash command.

  • Unsupported Environment

    While blender is portable, if you compile on a less common system, such as NetBSD, it may need minor edits to compile.

    Unless you want to spend time supporting les common development environments, it is best to use the default/standard development tools for your platform.

Reporting build problems

  • Use the Maillist or the #blendercoders IRC channel to report your problem.

    If using IRC, use an external pasting tool for posting more than 5 lines of code.

  • ALWAYS include a full error log. Just saying "it failed" with 1-2 lines containing the error is not very helpful.

    To create a full error log, you will want to redirect output into a file. See:( Linux/MacOS, MS-Windows)
